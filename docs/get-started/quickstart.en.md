# Quickstart

Use this reading order when you need to work on the public docs quickly.

## For readers

1. Start at [Welcome](/docs/get-started/overview).
2. Open [Foundations](/docs/core-concepts/overview).
3. Use [Reference](/docs/reference/overview) when you need schemas or migration notes.

## For maintainers

1. Add or update docs pages under `docs/**`.
2. Keep bilingual entry pages paired with `lang: en` and `lang: zh`.
3. Update `docs/navigation.en.yaml` and `docs/navigation.zh.yaml` when the public outline changes.
4. Keep legacy service docs linked, not duplicated, unless there is a clear migration target.
