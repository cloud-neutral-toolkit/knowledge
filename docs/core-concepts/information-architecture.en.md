# Information architecture

The public docs should read like a product manual, not a directory listing.

## Primary groups

- **Get started** for first contact
- **Foundations** for mental models
- **Integrations** for downstream service and repo links
- **Reference** for schemas and legacy maps

## Practical rule

When a page helps readers orient themselves, it belongs in one of these four groups even if the original source file sits somewhere else.
