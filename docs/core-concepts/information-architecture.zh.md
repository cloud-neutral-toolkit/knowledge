# 信息架构

公开文档应该像产品手册，而不是裸露的目录列表。

## 主导航分组

- **开始使用**：第一次进入时看什么
- **基础认知**：建立共同理解
- **集成关系**：连接下游服务与仓库
- **参考资料**：规范、矩阵与历史映射

## 实际规则

只要一个页面的作用是帮助读者建立定位，就应该被编排进这四组之一，而不是被动地跟随源文件原始目录。
